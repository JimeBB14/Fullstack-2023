# Sequence Diagram for Creating a New Note in Single-Page App

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: user presses the submit button
    Note right of browser: New note details are in HTTP request body, in JSON format
    Note right of browser: New note is added to local list that is then used to refresh shown notes

    browser->>server: HTTP POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    activate server
    server-->>browser: HTTP 201 resource created
    deactivate server

    Note right of browser: notice how the full list of notes is not retrieved anymore
